# Linux Commands for DevOps Engineers (Part 1)

## 1. `ls` — List Directory Contents

### Purpose
Used to view files and directories inside a location.

### Syntax

```bash
ls [options] [directory]
```

### Common Options

```bash
ls -l      # long listing format
ls -a      # show hidden files
ls -lh     # human readable sizes
ls -lrt    # latest modified files at bottom
```

### Example

```bash
ls -lh /var/log
```

### Real-Time DevOps Use Case

Checking log files, deployment artifacts, or verifying generated files after CI/CD execution.

```bash
ls -lrt /opt/tomcat/logs
```

### Interview Explanation

`ls` is one of the most frequently used Linux commands for file inspection. In DevOps, it helps verify deployments, inspect logs, validate permissions, and troubleshoot server-side issues quickly.

### Example


`pwd` is important in shell scripting and automation because many deployment scripts depend on correct execution paths.

# 5. `rm` — Remove Files or Directories

### Purpose

Deletes files and directories.

### Syntax

```bash

```

### Examples

```bash

rm -rf old_release/
```

`cp` is commonly used in deployment automation and backup strategies. Using backups before modifications is considered a best practice in DevOps.

# 9. `grep` — Search Text Patterns

## Purpose

`grep` is used to search specific patterns, keywords, or strings inside files, logs, and command outputs.

It is one of the most important troubleshooting and monitoring commands for DevOps engineers.

---

# Syntax

```bash
grep [options] "pattern" file_name
```

---

# Most Commonly Used Options

```bash
grep -i     # Ignore case sensitivity
grep -v     # Show lines NOT matching pattern
grep -n     # Show line numbers
grep -r     # Recursive search inside directories
grep -c     # Count matching lines
grep -w     # Match whole word only
grep -A 5   # Show 5 lines after match
grep -B 5   # Show 5 lines before match
grep -C 5   # Show 5 lines before and after
grep -E     # Extended regex
grep --color # Highlight matched pattern
```

---

# Basic Examples

## Search a word inside a file

```bash
grep error app.log
```

---

## Ignore case sensitivity

```bash
grep -i failed application.log
```

Matches:
- FAILED
- failed
- Failed

---

## Search recursively inside directories

```bash
grep -r "database" /etc/
```

Useful for finding configuration entries across multiple files.

---

## Show line numbers

```bash
grep -n "timeout" nginx.conf
```

Output Example:

```bash
45: proxy_read_timeout 60;
```

---

## Count occurrences

```bash
grep -c "ERROR" app.log
```

---

## Invert match

Show lines that do NOT contain a pattern.

```bash
grep -v INFO application.log
```

---

## Search multiple patterns

```bash
grep -E "ERROR|FAILED|CRITICAL" app.log
```

---

# Real-Time DevOps Use Cases

## 1. Checking Application Errors

```bash
grep ERROR catalina.out
```

Used while troubleshooting Java/Tomcat applications.

---

## 2. Monitoring HTTP Status Codes

```bash
grep "500" access.log
```

Finds Internal Server Errors from Nginx/Apache logs.

---

## 3. Kubernetes Pod Troubleshooting

```bash
kubectl logs pod-name | grep CrashLoopBackOff
```

Quickly filters important events from container logs.

---

## 4. Checking Failed Jenkins Builds

```bash
grep FAILURE jenkins.log
```

Useful during CI/CD troubleshooting.

---

## 5. Finding Unauthorized Login Attempts

```bash
grep "Failed password" /var/log/secure
```

Common Linux security monitoring activity.

---

## 6. Searching Environment Variables

```bash
env | grep JAVA
```

Used to verify runtime variables in servers and containers.

---

# Advanced DevOps Examples

## Find Large Error Spikes in Logs

```bash
grep ERROR app.log | wc -l
```

Counts total error entries.

---

## Real-Time Log Monitoring with grep

```bash
tail -f application.log | grep ERROR
```

Very common production monitoring command.

---

## Search Running Processes

```bash
ps -ef | grep nginx
```

Checks whether services are running.

---

## Filter Docker Containers

```bash
docker ps | grep redis
```

---

## Search Open Ports

```bash
netstat -tulnp | grep 8080
```

Checks which process is using port 8080.

# Pro Tips for Experienced DevOps Engineers

## Highlight Matches

```bash
grep --color "ERROR" app.log
```

---

## Search Exact Word Only

```bash
grep -w "root" /etc/passwd
```

Avoids partial matches.

---

## Show Context Around Errors

```bash
grep -C 3 ERROR app.log
```

Displays 3 lines before and after the matched line.

Very useful during production debugging.

# 10. `find` — Search Files and Directories

### Purpose

Finds files/directories based on conditions.

### Syntax

```bash
find [path] [options]
```

### Examples

```bash
find /var/log -name "*.log"
find / -type f -size +100M
```

### Real-Time DevOps Use Case

Finding large files causing disk issues or locating configuration files across servers.

```bash
find / -type f -name "application.properties"
```

### Interview Explanation

`find` is extremely useful for automation, cleanup jobs, backup verification, and troubleshooting storage-related production issues.
